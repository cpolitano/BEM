BEM

an object-oriented approach to CSS created through naming conventions for HTML and CSS classes

Why BEM?

  1. Faster
  2. Maintainable
  3. Extensible

BEM 101

  1. Everything is a class.
  2. Nothing is nested.

BEM looks like this

.block

.block__element

.block--modifier

Never do this.

Nest CSS rules (ruins flat specificity chain)

Override styles of an unrelated block

Make an unnecessary parent block if the child element can stand on its own
